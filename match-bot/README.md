# TactIQ — بوت المباريات (Match Content Bot)

بوت يراقب المباريات ويولّد تنبيهات + برومتات تصاميم + تحليلات على تيليغرام. بروح تطبيق TactIQ (تكتيك).

## المفاتيح (بملف `../.env`)
- `FOOTBALL_API_KEY` — من api-football (مجّاني ١٠٠ طلب/يوم؛ PRO ~$19 للتشغيل الدائم).
- `TELEGRAM_BOT_TOKEN` + `TELEGRAM_CHAT_ID` — بوت `@tactiq_alerts_bot`.
- (اختياريّ) `FINAL_CITY` — مدينة النهائي (مثلاً New York) لتصاميم التأهّل.

## الملفّات والأوامر

### المراقبة الحيّة
- **`find-fixture.js`** — يلاقي رقم مباراة: `node find-fixture.js 2026-07-18 France`
- **`watch.js`** — يراقب مباراة حيّة ويبعت تنبيهات: `node watch.js --fixture=ID --interval=4 --send`
  - `--interval=N` دقايق (توفير طلبات) · `--once` فحص مرّة · بلا `--send` = طباعة فقط.
  - يوقف لحالو بنهاية المباراة. الحالة محفوظة بـ`state/`.

### القرار والتنبيهات
- **`content-rules.json`** — قوائمك: `starPlayers` (النجوم الكبار)، `bigClubs`، `rivalries`، `priorityLeagues`، `alwaysCoverTeams`. عدّلها وقت ما بدّك.
- **`decide.js`** — يصنّف الحدث: `node decide.js [event.json]`
  - **aiDesign** (تصميم AI): نجم كبير سجّل · تأهّل/بطل · تذكير قمّة.
  - **resultCard**: لاعب عاديّ بمباراة مهمّة.
  - **skip**: ثانويّ.
- **`notify.js`** — يبني برومت Gemini + كابشن حسب الحدث (اختبار: `node notify.js event.json --send`).
  - أنواع: `goal` / `winning_goal` / `red_card` / `final` / `advance` / `champion` / `preview`.
  - يفرض **الواقعيّة** (منع كرتون/رسم) + يطلب **رفع صور مرجعيّة** (وجه اللاعب + القميص الحاليّ) لحلّ قِدَم بيانات Gemini.

### تذكير القمم الصباحيّ
- **`preview.js`** — قمم اليوم (كلاسيكو/ديربي/أبطال): `node preview.js --send` (شغّلو الصبح — Task Scheduler / cron).

### كرت النتيجة
- **`gen-card.js`** + `match-card.html`: `node gen-card.js match.json` → PNG + كابشن.

### خريطة الهدف التكتيكيّة (شبه يدويّ)
- **`goal-editor.html`** — محرّر بصريّ (افتحو بالمتصفّح): اسحب لاعبين + ارسم أسهم + شروحات → صدّر `play.json`.
- **`gen-play.js`** + `goal-map.html`: `node gen-play.js play.json` → أنيميشن. الـMP4 بتسجّلو أنت (screen record).

### تحليلات (بيانات حقيقيّة)
- **`shotmap.html`** + **`gen-shotmap.js`**: خريطة تسديدات + xG. `node gen-shotmap.js shots.json`.
  - مصدر البيانات: Understat (اختياريّ لاحقًا) أو JSON يدويّ.
- **`analyze.js`** — تحليل المدرّبين: `node analyze.js --fixture=ID --send` → يسحب إحصائيّات حقيقيّة → برومت تحليل Gemini + كابشن.

## أنواع التصاميم (الخلاصة)
| المناسبة | النوع | تلقائيّ؟ |
|---|---|---|
| 🌅 صباح القمّة | تذكير مواجهة (preview) | ✅ |
| ⚽ نجم كبير يسجّل | تصميم لاعب AI (+ صور مرجعيّة) | نصف (Gemini يدويّ) |
| 🎉 فريق يتأهّل/بطل | تصميم فريق (TO THE FINAL / CHAMPIONS) | نصف |
| 🅱️ لاعب عاديّ / دوري | كرت نتيجة | ✅ |
| 📊 نهاية مباراة | خريطة تسديدات + xG | ✅ (مع بيانات) |
| 🧠 تحليل | برومت تحليل + كابشن | ✅ سحب · Gemini يدويّ |
| 🎬 هدف مهمّ | خريطة هدف متحركة | يدويّ (دقيقة بالمحرّر) |

## سير العمل
1. `watch.js` يكشف حدث → تنبيه + برومت على تيليغرام.
2. للنجوم: تاخد البرومت → **ترفع صورة اللاعب + القميص** بـGemini → تولّد → تنشر.
3. للنتائج/التحليل: بيوصلك كرت/برومت تحليل جاهز.
4. للأهداف المهمّة: تفتح `goal-editor.html`، تبني اللقطة، تسجّل الأنيميشن.

## القاعدة الذهبيّة
- **تصميم AI للنجوم الكبار فقط** (الشبه بيزبط ويستاهل). الباقي كرت/تحليل.
- **واقعيّ دايمًا** (مش كرتون) + **صور مرجعيّة** للوجه والقميص الحاليّ.
- **مفاهيم متنوّعة مربوطة بالحدث** (مش تكرار احتفال).

## التشغيل الدائم لاحقًا
- GitHub Actions (cron) + باقة API-Football PRO (~$19) → مراقبة تلقائيّة بلا حاسوبك.
